# コーディング規約

継続的にメンテナンスしていくための、本リポジトリ共通のルール。
Python を対象とし、基本は [PEP 8](https://peps.python.org/pep-0008/) に準拠する。

## 言語・文字

- **識別子（変数・関数・クラス・定数名）は英語**で書く。ローマ字日本語（例: `buttin`）は使わない。
- **コメント・docstring は日本語**でよい。意図や背景（なぜそうしたか）を書く。
- ファイルは UTF-8、末尾は改行 1 つ。

## 命名規則

| 対象 | 規則 | 例 |
|---|---|---|
| 変数・関数 | `snake_case` | `random_reply_rate`, `strip_mentions()` |
| 定数（モジュール定数） | `UPPER_SNAKE_CASE` | `MARKOV_MIN_LEN` |
| クラス | `PascalCase` | `Markov`, `LLMFallback` |
| モジュール内部だけで使う名前 | 先頭にアンダースコア | `_read_int()`, `_IGNORED_TOKENS` |
| 環境変数 | アプリ接頭辞 + `UPPER_SNAKE` | `MUNO_RANDOM_REPLY_RATE` |

- **省略形を避ける**。意味が一目で分かる完全な単語を使う
  （`cands` → `candidates`、`nxt` → `next_word`、`kw` → `keyword`）。
- ただし**そのドメインで標準的な略語は可**（n-gram の `w1, w2, w3` など）。
- 真偽値を返す関数・真偽値変数は `is_` / `has_` / `should_` で始める（`should_reply`）。
- 名前で型を表す（リストは複数形 `responses`、辞書は `〜_by_〜` など）。

## 構造・スタイル

- インデントは**スペース 4 つ**。1 行は概ね 100 文字以内。
- **型ヒントを付ける**（特に公開関数の引数・戻り値）。
- 公開モジュール・公開関数・クラスには **docstring** を付ける。
- import は「標準ライブラリ → サードパーティ → 自プロジェクト」の順にグループ分けする。
- マジックナンバー・固定文字列は**モジュール定数に切り出す**。
- 1 関数 1 責務。長くなったら分割する。

## エラー処理

- 例外は握りつぶさない。握りつぶす場合は**理由をコメントで書く**
  （例: LLM フォールバックが落ちても本体は止めない）。
- 失敗時に何を返すか（`None` / 空文字 / 例外）を docstring に明記する。

## セキュリティ（公開リポジトリ前提）

- **秘密情報をコードに直書きしない**。トークン・API キーは環境変数（`.env`）から読む。
- `.env` と学習データ（`data/*.sqlite3`）は `.gitignore` で除外する。
- 設定例は `.env.example` に**空の値**で置く。
- 外部入力（ユーザー発言）はそのまま `eval`/`exec`/SQL 文字列連結に渡さない
  （SQL はプレースホルダを使う）。
